SQL vs NoSQL

🗄️

SQL (Relational)

Structured data with defined schemas. ACID transactions, complex JOINs. Best for financial data, user records, and structured business logic.

NoSQL (Non-Relational)

Flexible schemas, horizontal scaling, high availability. Document, key-value, graph stores. Best for unstructured data and rapid iteration.

SQL in Practice

PostgreSQL
-- Create a normalised users table CREATE TABLE users ( id SERIAL PRIMARY KEY, email VARCHAR(255) UNIQUE NOT NULL, name VARCHAR(100) NOT NULL, created_at TIMESTAMP DEFAULT now() ); -- Index for fast email lookups CREATE INDEX idx_users_email ON users(email); -- JOIN with aggregation SELECT u.name, COUNT(o.id) AS orders, SUM(o.amount) AS revenue FROM users u LEFT JOIN orders o ON u.id = o.user_id WHERE o.created_at > '2024-01-01' GROUP BY u.id ORDER BY revenue DESC LIMIT 10;

Database Normalisation

1NF
First Normal Form
Eliminate duplicate columns. Each cell holds atomic values. Every row is unique.
2NF
Second Normal Form
Meet 1NF. Remove partial dependencies — every non-key attribute depends on the whole primary key.
3NF
Third Normal Form
Meet 2NF. Remove transitive dependencies — non-key fields must not depend on other non-key fields.
